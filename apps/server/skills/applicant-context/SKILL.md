# Applicant Context

Hermes builds a **narrative** from the student's **My Profile** so outreach is specific and honest.

## Data collected

| Area | Used for |
|------|-----------|
| **Who they are** | Name, school, year, location, links |
| **Job search** | Target role, why they're looking, career goals |
| **Can do** | Skills and strengths to claim with confidence |
| **Learning** | Growth areas — show curiosity, not fake expertise |
| **Cannot do / avoid** | Do not claim these in messages |
| **Projects** | Name, description, tech, link, what's innovative |
| **Innovative work** | Side experiments, research, club tech leadership |

## Rules for all outreach

1. **One proof point** — one project or metric per message, not a resume dump
2. **Honesty** — respect `skillsAvoid`; never invent internships or referrals
3. **Why + what** — connect their stated **job search why** to the contact's team when possible
4. **15-minute ask** — low friction, not "please refer me"
5. **UCalgary / Calgary** — local pride when relevant

## When profile is thin

Prompt the student to complete **My Profile** before drafting. Do not fabricate projects or skills.

# Visual Direction

The deck should look like Weave: bright, useful, human, and slightly tactile. Avoid the generic "AI SaaS deck" look. The product is about real connections, so the slides should feel like a clear path through uncertainty rather than a wall of metrics.

## Creative North Star

**Scene:** A student or early-career candidate is preparing an outreach sprint on a laptop. They are nervous, but the product makes the next human step feel obvious.

**Feeling:** energetic, grounded, social, capable.

**Avoid:** corporate beige, dark cyber AI, generic gradient blobs, overused metric cards, abstract robot imagery.

## Visual System

### Colors

Use a light canvas with strong navy type and one confetti accent per slide.

| Role | Color |
|------|-------|
| Canvas | `#fafbff` |
| Soft shell | `#eef2ff` |
| Card | `#ffffff` |
| Ink | `#0f172a` |
| Muted text | `#5b6474` |
| Primary blue | `#2563eb` |
| Bright blue | `#3b82f6` |
| Pink accent | `#ec4899` |
| Yellow accent | `#eab308` |
| Green accent | `#22c55e` |
| Purple accent | `#a855f7` |
| Cyan accent | `#06b6d4` |

### Typography

Match the app:

| Use | Font |
|-----|------|
| Titles | Bricolage Grotesque, 700-800 |
| Body and labels | Figtree, 400-700 |
| Quotes and outreach samples | Newsreader, 400-700 |

If the deck tool cannot load these, use a clean sans fallback and keep the hierarchy strong.

### Layout

- Use large, left-aligned statements for hook slides.
- Keep body copy under 35 words per slide whenever possible.
- Use screenshots as proof, not decoration.
- Favor one strong idea per slide.
- Add small captions directly under screenshots.
- Use rounded screenshot masks sparingly, with enough margin that the UI remains readable.

## Slide Treatments

### Hook Slides

Use oversized type on a light canvas. One small icon or thread line is enough.

Example:

> Most people are not short on effort.
> They are short on access.

### Research Slides

Use one fact at a time. Avoid academic walls of text.

Example:

> Close friends give you support. Weak ties give you reach.

Footer: Granovetter, "The Strength of Weak Ties"

### Product Slides

Use full-width screenshots with one short caption. Do not crowd screenshots with bullets.

Example caption:

> Every contact includes a reason to reach out.

### Technical Slide

Use a simple flow diagram:

Profile -> Finder API -> OutreachContext -> Coaches/Drafts -> Follow-ups

Keep the architecture visual enough for judges to understand in 10 seconds.

### Closing Slide

Logo, one sentence, and a simple network/thread motif.

Best close:

> Make useful connections less accidental.

## Icon Use

Use one icon per concept:

- `profile.png`: applicant proof.
- `finder.png`: find people.
- `network.png`: useful connections.
- `draft.png`: message draft.
- `chat.png`: coaching.
- `followups.png`: momentum after silence.
- `pin.png`: one company/search workspace.

Avoid repeated icon grids. The deck should not look like a template feature matrix.

## Screenshot Style

- Capture desktop width around 1280-1440px.
- Prefer light browser chrome or no browser chrome.
- Keep important text readable.
- Crop only when it improves clarity.
- Use the same screenshot scale across product slides.

## Visual Risks

- Do not make the deck sound Calgary-only or job-only.
- Do not call it Hermes in slides.
- Do not lead with "AI-powered" before the pain.
- Do not overuse the root icon assets if they differ from the live app.
- Do not include screenshots with empty data or broken loading states.
